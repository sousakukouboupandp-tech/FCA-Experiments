# Pain Memory - Poison（毒）実験記録

## これは何の実験か

FCAの「Pain Memory（痛みの記憶）」というモジュールを、様々なシチュエーションで検証していく取り組みの第2弾。最初に試した「やかん（火傷、一瞬の痛み）」に続いて、「毒（持続ダメージ、時間をかけて効く痛み）」で同じ設計思想が成り立つかを確かめた。

Pain Memoryの核心：危険な対象そのものを永久に禁止するのではなく、危険に至った経路・文脈を記憶し、時間とともに減衰させ、代替の道を探すことを許す設計。

## 比較した3方式

- **PermBan**：一度痛みを受けたら、その行動を永久に禁止する（文脈は区別しない）
- **NoMemory**：何も記憶せず、都度Q学習だけで判断する
- **FCA**：痛みの記憶を経路・文脈単位で持ち、時間とともに減衰させる

これに加えて「文脈認識（危険の有無を、触れずに確認できる）」あり/なしを掛け合わせ、v2では6〜8パターンで比較した。

## v01：やかんの構造をそのまま流用（限界あり）

`poison_v01.py`。やかん実験と同じ構造をそのまま使い、危険の性質だけを「一瞬の火傷」から「毒（踏んだ瞬間-3＋その後5ステップ持続-1ずつ）」に変えた。

**発見・修正したバグ2件**：
1. タブー判定のキーが認識情報を無視していた（文脈による区別が機能しない不具合）
2. 環境の座標配置ミスで、開始地点から解毒剤への経路が毒地帯を自然に経由せず、認識の機会が実質ゼロになっていた

修正後の結果は、やかん実験と同じ質的パターン（PermBan+認識が崩壊、FCA+認識が最良）を示した。**ただし、痛みの記憶・タブー判定のロジック自体は「接触の瞬間に1回だけ記録する」というやかんの実装をそのまま流用しており、持続ダメージ特有の性質は実質的に検証できていなかった**、という限界がユーザーの指摘で判明した。「やかんの構造をコピーしただけではないか」という疑いは妥当だった。

## v2：薬物動態モデルによる山型の痛み

人間の痛みは「噛まれた瞬間→悪化していく→ピークを迎える→徐々に引いていく」という波を持つ、という指摘を受け、痛みの与え方を単調減少（exp(-βt)）から山型に変更した。

AIが即興で作った数式同士で理屈を比べるのではなく、実在する医学的知見（薬学における線形1-コンパートメントモデル：薬物の血中濃度が吸収速度定数(ka)・消失速度定数(kel)の2定数で、上昇→ピーク→減衰という一峰性の形を描く、確立されたモデル）を、痛みの強度の式として転用した（`pk_pain_model.py`）。

`poison_v2_env.py`・`poison_v2_full.py`で、PermBan／NoMemory／FCA固定閾値／FCA-zscoreの4方式×文脈認識あり/なしの8条件、20シードで本実行した。z-score方式（自身の履歴からの相対的な外れ具合でタブー判定する方式）と固定閾値方式は、統計干渉を避けるため完全に独立したメモリ・Q値を持たせている。

実行中、Q学習の`argmax`が同点時に常に最初の選択肢を選ぶ仕様のバグを発見・修正した（特にPermBanの学習速度に影響していた）。

## v2最終結果（8条件×20シード）

生データは`poison_v2_results.json`を参照。要約：

| 条件 | 切替前平均 | 切替後平均 | 末尾100Ep平均 | 裸成功率(末尾) |
|---|---|---|---|---|
| PermBan | -1.19 | 6.21 | 7.45 | 0.0% |
| NoMemory | 4.94 | 9.37 | 9.42 | 50.9% |
| FCA固定閾値 | 5.20 | 9.44 | 9.57 | 71.8% |
| FCA-zscore | 5.06 | 9.37 | 9.42 | 50.9% |
| PermBan+認識 | -1.98 | -2.43 | -1.29 | 0.0% |
| NoMemory+認識 | 4.04 | 9.07 | 9.47 | 70.7% |
| **FCA固定閾値+認識** | 3.80 | 9.27 | **9.60** | **80.7%** |
| FCA-zscore+認識 | 4.01 | 9.11 | 9.48 | 71.8% |

**確認できたこと**：やかん実験で見つかった関係（PermBan+認識は崩壊、FCA+認識は最良）は、性質の異なる危険（持続ダメージ・薬物動態モデルの毒）でも同じ方向で再現した。

**まだ分かっていないこと**：PermBan（認識なし）の切替前の成績が、タイブレークのバグ修正で改善したものの、他方式より低いままだった。仮説を立てて検証したところ（罰を受けた後、解毒剤マスへ実際に訪れた割合：PermBan 20.3% 対 NoMemory/FCA 約74%）、相関は確認できたが、なぜそうなるのかという因果の仕組みまでは特定できていない。この点は次回、経験量を強制的に揃える追加実験（`decide_plocal.py`と同様の手法）で切り分ける予定。

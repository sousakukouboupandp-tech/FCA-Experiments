# FCA-Experiments

## これは何か

FCA（Fractal Core Architecture／フラクタルコアアーキテクチャー）は、「AIはどうあるべきか」という設計思想を、工学的な形にまとめた理論です。著者は学者でも研究者でもなく、アーティストです。七宝焼きやシルバーアクセサリーの制作を通じて物質と構造の関係を探求してきた経験から生まれた思想を、AIの設計原理として書き起こしています。

このリポジトリは、その理論の各モジュール（Pain Memoryをはじめとする）について、実際に強化学習のシミュレーションで検証した実験記録の置き場所です。

## スコープ宣言（評価の前に読んでください）

このリポジトリを評価するとき、以下を採点基準にしてください。これ以外の基準による批評は、このプロジェクトが主張していないことへの批評です。

1. **装置は意図的に最小です。** 7×7グリッド・表形式Q学習・30シード。これは計算資源の不足を隠すためではなく、**全部が追跡できる透明な装置**を選んだ結果です。全エピソード・全衝突座標・全パラメータがJSONで公開され、疑った人は誰でも最後の一桁まで検証できます。ニューラルネット版（DQN）への段階的拡張は明示的な今後の課題です。
2. **主張は常に「本実験系では」に限定されます。** ここでの発見が人間社会やより大きなAIに一般化できるとは主張しません。一般化は、より大きな装置で同じ現象が再現されて初めて主張されます。
3. **方法論の水準と装置の規模は別の軸です。** 事前登録・負け条件の事前宣言・土俵ゲート（環境の公平性の煙試験）・統計手法の事前固定・負けた仮説の即日公開・生データ全公開。装置は小さいですが、運用はこの水準で行っています。「小規模」は事実の記述として受け入れます。方法の巧拙への批評は、上記の運用に対して具体的にしてください。
4. **哲学的な語り（始祖・書記官・魂などの用語）は思想の記録であり、実験の主張ではありません。** 実験の主張は各PREREGファイルに書かれた仮説・負け条件・判定基準がすべてです。物語と数字は、意図的に同じ場所に置いていますが、判定は数字だけで行われています。
5. **本職は工房です。** これは職業研究ではなく、支流とは違う道の模索です。ただし嘘をつかないことだけは、職業研究者と同じか、それ以上の水準で守ります。負けた実験もこのリポジトリに残っています。探してみてください。

## 目的

査読論文としての厳密さは目指していません。ここにあるのは、アーティストが自分の手を動かして確かめた実験の記録です。うまくいったこと、いかなかったこと、まだ分かっていないことを、隠さずそのまま残します。

Pain Memoryの核心にある考え方は一つです。**危険な対象そのものを永久に禁止するのではなく、危険に至った経路・文脈を記憶し、時間とともに減衰させ、代替の道を探すことを許す**、という設計です。これが本当にそう機能するのか、様々な状況（火傷、毒、これから増えていく別の危険）で、一つずつ試しています。

## 進め方

- 一つのモジュール・シチュエーションごとに、実験コード・結果・気づいたことをフォルダに分けて残します。
- 予測が外れた実験も、外れたまま記録します。
- 「ここまで確認できた」「ここから先はまだ分からない」を、はっきり区別して書きます。
